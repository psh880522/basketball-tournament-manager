```
[1] tailwind.config.ts → tertiary 토큰 추가
      ↓ (토큰 준비 후)
[2] app/globals.css → .arena-gradient 추가
      ↓ (클래스 준비 후)
[3] components/nav/GlobalHeader.tsx → 헤더 스타일
      ↓
[4] components/nav/NavMenu.tsx → 버튼 색상 (헤더와 세트)
      ↓
[5] components/layout/LandingFooter.tsx → 신규 생성
      ↓
[6] app/(public)/page.tsx → 랜딩 페이지 전면 재작성

[7] app/(public)/layout.tsx → LandingFooter 연결
```

**의존 관계 요약**:
- `[1][2]`는 독립적, 동시 진행 가능
- `[3][4]`는 `[1][2]` 완료 후 진행 (클래스 참조)
- `[5]`는 독립적으로 진행 가능
- `[6]`은 `[1]~[5]` 완료 후 진행 (모든 스타일/컴포넌트 준비 필요)

## 6. 리스크 및 주의사항

### 6-1. Suspense fallback 스타일 불일치

`<OpenTournamentsList>`, `<InProgressTournamentsList>` 내부의 `fallback` 텍스트가 새 레이아웃과 어색할 수 있음.

- **대응**: fallback을 단순 텍스트 대신 skeleton 카드 형태로 변경 권장
  ```
  fallback: <div className="h-32 rounded-xl bg-gray-100 animate-pulse" />
  ```

### 6-2. 히어로 섹션 텍스트 하드코딩

현재 `"농심 3X3 농구대회에 오신 것을 환영합니다!"` 고정. 타이포 변경 후 줄바꿈 지점이 달라질 수 있음.

- **대응**: `<br />`을 명시적으로 사용하거나 `whitespace-pre-line` 활용

### 6-3. `GlobalHeader` minimal prop 미사용 경로

`GlobalHeader`에 `minimal` prop이 있음 (인증 레이아웃에서 사용). 로고 스타일 변경 시 `minimal` 분기도 함께 수정 필요.

```
// GlobalHeader.tsx line 17-24
if (minimal) {
  return (
    <header ...>
      <Link href="/" className="text-base font-bold tracking-tight text-slate-900">
        🏀 23Board   ← 이 부분도 함께 스타일 변경
```

### 6-4. `components/ui/Card.tsx` 미사용 처리

카드 레이아웃을 `<Card>` 컴포넌트 대신 커스텀 `<div>` + `border-l-4`로 교체하면, `Card` 컴포넌트가 랜딩 페이지에서 더 이상 사용되지 않음.

- **대응**: `Card` 컴포넌트는 다른 곳에서도 사용 중이므로 삭제하지 않음. 랜딩 페이지에서만 미사용 처리.

### 6-5. Tailwind CSS 4 purge 주의

`arena-gradient` 클래스를 `globals.css`에 추가할 경우 Tailwind 4의 content 스캔 대상 밖에 정의되므로 purge 걱정 없음. 단, `tailwind.config.ts`의 `tertiary` 토큰은 실제로 사용되는 클래스(`border-tertiary`, `text-tertiary` 등)가 없으면 번들에 포함되지 않을 수 있음.

- **대응**: `tertiary` 토큰 추가 후 반드시 실제 클래스에서 참조해야 번들 포함됨

### 6-6. 로그인 레이아웃(Sidebar)과의 시각적 격리

랜딩 페이지의 히어로 배경이 강렬해지면, 로그인 후 동일 페이지(`/`)에서 `Sidebar` 레이아웃으로 전환될 때 시각적 불일치가 발생할 수 있음.

- `app/(public)/layout.tsx`에서 로그인 시 Sidebar 레이아웃을 사용하므로, 히어로 섹션이 그대로 노출됨
- **대응**: 히어로 섹션을 비로그인 상태에서만 노출하는 분기 추가 검토
  ```
  // page.tsx
  {!isLoggedIn && <HeroSection />}
  ```
  → 단, 이는 기존 로직 변경이므로 구현 시 결정 필요

### 6-7. (v2 추가) 히어로 배경 이미지 — `public/logo.jpg` 임시 사용

`public/logo.jpg`는 로고 이미지이므로 히어로 배경으로 어울리지 않을 수 있음. 단, `opacity-20` + `mix-blend-luminosity`로 흐리게 처리하므로 시각적으로는 허용 가능한 수준.

- **대응**: `HERO_BG_IMAGE` 상수로 분리하여 추후 적합한 농구 코트/경기 사진으로 교체 용이하게 구성
- 교체 시 `public/` 폴더에 이미지 추가 후 상수값만 변경

### 6-8. (v2 추가) 역할별 서브메시지 UX

별도 파란 카드 제거 후 user 역할 안내가 히어로 서브메시지로만 표현됨. 정보 밀도가 줄어드는 점은 트레이드오프.

- 원래 파란 카드에 있던 상세 안내문("이름, 연락처 등 기본 정보를 입력하고...") 은 히어로에서 한 줄로 축약됨
- 상세 안내는 `/onboarding/profile` 페이지로 이동 후 확인 가능하므로 UX상 문제없음
