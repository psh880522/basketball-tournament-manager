## 5. 트레이드오프 및 고려사항

### 5.1 병합 시 정렬 기준

`scheduled_at` 기준 오름차순 정렬을 적용한다. 현재 리그/토너먼트 각 테이블은 API에서 반환된 순서 그대로 표시하고 있으므로, 병합 후 정렬이 오히려 **더 명확한 시간 순서**를 제공한다.

- `scheduled_at`이 없는 경기(미배정)는 정렬 마지막에 위치 (`Infinity` 처리)
- 동일 시간이면 `id` 알파벳순으로 안정 정렬

### 5.2 리그와 토너먼트 경기가 혼재 시 "구분" 열 역할

기존에는 리그/토너먼트를 별도 테이블로 분리했기 때문에 "구분" 열이 각각 그룹명·라운드명만 표시해도 문맥상 명확했다. 병합 후에는 "구분" 열이 둘을 구분하는 유일한 시각적 단서가 된다.

| 경기 종류 | 구분 열 예시 |
|---------|-----------|
| 리그 경기 | `조별 1` |
| 토너먼트 경기 | `8강 1경기` |

포맷터(`formatTournamentCategoryLabel`, `m.groupName`)의 출력값이 이미 충분히 구별된다고 판단. 별도 배지나 색상 표시는 추가하지 않는다 (TODO2: 내용 변경 금지).

### 5.3 Empty State 처리

기존:
- 리그 경기 없음 → "리그 경기가 없습니다."
- 토너먼트 경기 없음 → "토너먼트 경기가 없습니다."

병합 후:
- `allMatches.length === 0` 이면 별도 empty state 없음 (디비전 자체가 matches 없이 생성되지 않으므로 실질적으로 발생 안 함)
- 기존 empty state `<p>` 2개 모두 제거
