## 1. 파일 참조 반영 요약

### research.md 에서 채택한 기술 스택

| 채택 내용 | 이유 |
|----------|------|
| Next.js App Router, Server Component, `force-dynamic` | 기존 대시보드 패턴 그대로 유지. 클라이언트 상태 없이 서버 렌더링으로 충분 |
| `Promise.all()` 병렬 데이터 조회 | 6개 API 호출을 직렬로 쌓으면 TTFB가 선형 증가 — 기존 패턴 확장 |
| 기존 `lib/api/matches.ts` 파일에 함수 추가 | 이미 `MatchListRow` 타입과 `scheduled_at` 컬럼 활용 코드가 존재 → 신규 파일 불필요 |
| Tailwind CSS + 기존 CSS 변수(`--ui-accent`, `--ui-bg`) | 추가 설치 없이 디자인 시스템 완결 |
| `lucide-react` 설치 필요 | 현재 `package.json`에 없음 — 설치 후 사용 |
| RLS 추가 마이그레이션 없음 | 읽기 전용 조회 위주, `createSupabaseServerClient()` 세션 기반 RLS로 충분 |

async function getMyDashboardSummary(): Promise<ApiResult<DashboardSummary>> {
  // 1. team_members WHERE user_id = me → teamCount, captainTeamCount

  //    → activeApplicationCount

  //             AND scheduled_at BETWEEN now() AND now()+7d
  //    → thisWeekMatchCount

#### `getMyPendingActions()` — `lib/api/dashboard.ts`

```typescript
// 인증: 세션 필수
// 반환: ApiResult<PendingAction[]>

async function getMyPendingActions(): Promise<ApiResult<PendingAction[]>> {
  // captain인 팀만 대상

  // Action 1: payment_pending 신청
  //   SELECT * FROM tournament_team_applications
  //   WHERE team_id IN (captain teams) AND status='payment_pending'
  //   → urgency: "high"

  // Action 2: confirmed 신청 중 roster 비어있는 것
  //   SELECT tta.*, r.id as roster_id, COUNT(rm.id) as member_count
  //   FROM tournament_team_applications tta
  //   LEFT JOIN rosters r ON r.application_id = tta.id
  //   LEFT JOIN roster_members rm ON rm.roster_id = r.id
  //   WHERE tta.team_id IN (captain teams) AND tta.status='confirmed'
  //   GROUP BY tta.id, r.id
  //   HAVING COUNT(rm.id) = 0
  //   → urgency: "medium"

  // Action 3: team_join_applications pending 건수
  //   SELECT team_id, COUNT(*) FROM team_join_applications
  //   WHERE team_id IN (captain teams) AND status='pending'
  //   GROUP BY team_id
  //   → urgency: "low"

  // 정렬: high → medium → low
}
```

## 6. 데이터베이스 스키마

> 신규 마이그레이션 없음. 기존 테이블 쿼리 패턴만 추가.

### 대시보드 기능에서 사용하는 테이블 관계

```
auth.users
  │
  ├── profiles (role: "player")
  │
  ├── player_profiles (display_name, position, career_level)
  │
  ├── team_members ─── teams
  │     ├── role_in_team: "captain" | "player"
  │     └── team_id → teams.id
  │
  ├── team_join_applications ─── teams
  │     └── status: "pending" | "approved" | "rejected"
  │
  └── (via team_members.team_id)
        │
        ├── tournament_team_applications ─── tournaments, divisions
        │     └── status: "payment_pending" | "paid_pending_approval"
        │                  | "confirmed" | "waitlisted" | "cancelled" | "expired"
        │
        ├── rosters ─── roster_members
        │     └── (application_id 1:1)
        │
        └── matches ─── tournaments, divisions, courts
              ├── team_a_id, team_b_id → teams.id
              ├── status: "scheduled" | "in_progress" | "completed"
              ├── scheduled_at: timestamptz (nullable)
              └── winner_team_id → teams.id (nullable)
```

### 쿼리 접근 방식 (RLS 확인 사항)

| 테이블 | 현재 RLS | 대시보드 필요 접근 | 대응 |
|-------|---------|-----------------|------|
| `team_members` | 팀원만 조회 가능 | 본인 소속 팀 조회 | ✅ 세션 기반 자동 |
| `tournament_team_applications` | 팀원 or 운영자 | 소속 팀 신청 조회 | ✅ 기존 함수 재사용 |
| `matches` | **확인 필요** | 소속 팀 경기 조회 | ⚠️ 확인 후 필요시 정책 추가 |
| `rosters` / `roster_members` | 팀원 조회 가능 | 로스터 인원 수 조회 | ✅ |
| `team_join_applications` | captain / 본인 | captain인 팀 pending 조회 | ✅ |
| `courts` | 전체 공개 | 코트명 JOIN | ✅ |
