## 2. Phase 2: 공통 인증 가드 추출

### 배경

`requireOrganizer()` / `ensureOrganizer()` 함수가 **3개 파일에 각각 private으로 선언**되어 있고, `requireOperationRole()`도 `results.ts`에 private으로 선언되어 있다.

| 파일 | 함수명 | 특이사항 |
|------|--------|---------|
| `lib/api/bracket.ts:80` | `requireOrganizer()` | 단순 버전 |
| `lib/api/schedule-slots.ts:434` | `requireOrganizer()` | 단순 버전, **15회 이상 내부 호출** |
| `lib/api/tournaments.ts:163` | `ensureOrganizer()` | 상태별 세분화 메시지 (`empty` 별도 처리) |
| `lib/api/results.ts:120` | `requireOperationRole()` | operation role 확인 |

### 신규 파일: `src/lib/auth/guards.ts`

```typescript
// src/lib/auth/guards.ts (Pseudocode)
import { getUserWithRole, isOperationRole, type Role } from "./roles"

type GuardSuccess = { ok: true; userId: string; role: Role }
type GuardFailure = { ok: false; error: string }
export type GuardResult = GuardSuccess | GuardFailure

const AUTH_ERRORS = {
  unauthenticated: "로그인이 필요합니다.",
  error: "인증 오류가 발생했습니다.",
  empty: "프로필이 없습니다.",
} as const

// organizer 전용 작업에서 사용
export async function requireOrganizer(): Promise<GuardResult> {
  const result = await getUserWithRole()
  if (result.status !== "ready") {
    return { ok: false, error: AUTH_ERRORS[result.status] ?? "인증 오류가 발생했습니다." }
  }
  if (result.role !== "organizer") {
    return { ok: false, error: "권한이 없습니다." }
  }
  return { ok: true, userId: result.user!.id, role: result.role }
}

// organizer + manager 허용 작업에서 사용 (결과 입력, 순위 계산 등)
export async function requireOperationRole(): Promise<GuardResult> {
  const result = await getUserWithRole()
  if (result.status !== "ready") {
    return { ok: false, error: AUTH_ERRORS[result.status] ?? "인증 오류가 발생했습니다." }
  }
  if (!isOperationRole(result.role)) {
    return { ok: false, error: "권한이 없습니다." }
  }
  return { ok: true, userId: result.user!.id, role: result.role! }
}
```

### 각 파일 변경 패턴

```typescript
// lib/api/bracket.ts
// BEFORE: 파일 내부에 private requireOrganizer() 선언 + getUserWithRole import
// AFTER:

// private 함수 제거, 호출 코드는 동일

// lib/api/schedule-slots.ts (동일 패턴, 15개 이상 호출처)
import { requireOrganizer } from "@/src/lib/auth/guards"

// lib/api/results.ts
import { requireOperationRole } from "@/src/lib/auth/guards"

// lib/api/tournaments.ts

// ensureOrganizer() 제거, 모든 호출을 requireOrganizer()로 교체
```

**페이지 레벨 `getUserWithRole()` 직접 호출은 변경하지 않는다.**
페이지에서는 role 값을 props로 전달하거나 redirect 처리가 목적이므로 guard 패턴과 용도가 다르다.
