### 4.1 `matches/page.tsx` — 소규모 CSS 수정

**현재 vs 변경 후 차이점:**

#### 변경 A: 코트 헤더 폰트 크기
```tsx
// Before
<h2 className="text-sm font-semibold text-gray-700">
  🏀 {section.label}
</h2>

// After
<h2 className="text-lg font-semibold">
  🏀 {section.label}
</h2>
```
> 이유: ScheduleSlotsBoard는 `text-lg font-semibold` 사용. `text-gray-700`은 디비전 헤더에서 사용하므로 코트 헤더에서 제거.

---

#### 변경 B: 섹션 타이틀 스타일 (리그 / 토너먼트 라벨)
```tsx
// Before (2곳: 리그, 토너먼트)
<p className="mb-2 text-xs font-semibold text-gray-500">
  리그
</p>

// After
<p className="mb-2 text-xs font-semibold uppercase tracking-wide text-slate-500">
  리그
</p>
```
> 이유: ScheduleSlotsBoard 섹션 타이틀은 `uppercase tracking-wide text-slate-500` 적용.

---

#### 변경 C: 리그 테이블 colgroup 너비 통일
```tsx
// Before (리그 테이블 colgroup)
<colgroup>
  <col className="w-28" />  {/* 시간 */}
  <col className="w-24" />  {/* 구분 ← w-24 */}
  <col className="w-auto" />
  <col className="w-24" />  {/* 상태 ← w-24 */}
  <col className="w-24" />  {/* 점수 ← w-24 */}
</colgroup>

// After
<colgroup>
  <col className="w-28" />  {/* 시간 */}
  <col className="w-28" />  {/* 구분 ← w-28로 통일 */}
  <col className="w-auto" />
  <col className="w-28" />  {/* 상태 ← w-28로 통일 */}
  <col className="w-28" />  {/* 점수 ← w-28로 통일 */}
</colgroup>
```
> 이유: ScheduleSlotsBoard의 고정 너비 열은 모두 `w-28`. 리그 테이블의 `w-24` 열들을 일치시킴.

---

#### 변경 D: th/td 패딩 통일
```tsx
// Before (th)
<th className="px-4 py-3">시간</th>

// After
<th className="px-3 py-2">시간</th>

// Before (td)
<td className="whitespace-nowrap px-4 py-3 text-gray-600">

// After
<td className="whitespace-nowrap px-3 py-2 text-gray-600">
```
> 이유: ScheduleSlotsBoard는 `px-3 py-2` (더 컴팩트). `px-4 py-3`은 상이함.
> 주의: 모든 `<th>`와 `<td>` (리그 + 토너먼트 테이블 양쪽) 적용 필요.

---

**변경 라인 집계**:
- 코트 헤더 1줄
- 섹션 타이틀 2곳 × 1줄 = 2줄
- 리그 colgroup 3열 = 3줄
- th 패딩: 리그 5개 + 토너먼트 5개 = 10줄
- td 패딩: 리그 5열 × N행 + 토너먼트 5열 × M행 → 각 `<td>` 클래스 수정
- **총 ~25-30곳 수정** (로직 변경 없음)

**StatusBadge 컴포넌트 추가**:
현재 bracket/tournament/page.tsx에는 StatusBadge가 없음. matches/page.tsx와 동일한 인라인 함수로 추가:
```tsx
function StatusBadge({ status }: { status: string }) {
  const colors: Record<string, string> = {
    scheduled: "bg-slate-100 text-slate-700",
    in_progress: "bg-amber-100 text-amber-800",
    completed: "bg-emerald-100 text-emerald-800",
  };
  return (
    <span className={`inline-block rounded-full px-2 py-0.5 text-xs font-semibold ${
      colors[status] ?? "bg-gray-100 text-gray-700"
    }`}>
      {STATUS_LABELS[status] ?? status}
    </span>
  );
}
```

## 5. 트레이드오프 및 고려사항

### 5.1 `table-fixed` 적용 시 텍스트 잘림

`table-fixed` + `w-28`(7rem) 는 고정 너비를 강제 적용한다. 긴 팀명이나 라벨이 있을 때 `truncate`가 없으면 잘릴 수 있음.

**현재 상태**: matches/page.tsx의 경기 열(`<td>`)에 `truncate` 클래스가 이미 있음 → 안전.
**브라켓 페이지**: 새로 작성하므로 경기 열에 `truncate` 추가 권장.

---

### 5.2 ResultForm의 스코어 입력 colgroup

ResultForm의 테이블은 스코어 입력 필드(`<input>`) 때문에 열이 8개로 독자적인 구조:
```
시간(w-28) / 구분(w-24) / 경기(w-auto) / 점수A(w-16) / 구분자(w-8) / 점수B(w-16) / 상태(w-24) / 저장(w-20)
```

이 colgroup은 **변경하지 않는다.** 입력 필드 너비에 맞춰 설계된 구조이므로 ScheduleSlotsBoard 패턴을 억지로 맞추면 입력 필드가 잘리거나 레이아웃이 깨질 수 있음.

**결정**: ResultForm 테이블 colgroup은 현행 유지. CSS 통일은 헤더 폰트 크기·섹션 타이틀에만 적용.

---

### 5.3 bracket/tournament/page.tsx의 기존 코드 스타일

현재 파일은 `renderTournamentMatch()` 라는 문자열 반환 함수로 경기 정보를 포맷팅:
```tsx
return `[${roundLabel}] ${seedLabel} (${scoreLabel})`;
```
이를 JSX 테이블 구조로 변환하면 각 정보가 별개 `<td>`로 분리된다. 기존 문자열 포맷 함수는 제거하고 각 cell에 직접 렌더링.

---

### 5.4 최상위 컨테이너 space-y

| 페이지 | 현재 | 통일 후 |
|-------|------|--------|
| matches/page.tsx | `space-y-6` | `space-y-4` |
| ResultForm.tsx | `space-y-6` | `space-y-4` |
| bracket/tournament | inline style | `space-y-4` |

ScheduleSlotsBoard는 `space-y-4` 사용. matches와 result는 현재 `space-y-6`이므로 `space-y-4`로 변경.
영향: 카드 간격이 약간 좁아짐 (24px → 16px).

---

### 5.5 브라켓 페이지의 디비전 선택 폼 스타일

현재 `<form>` 에 `style={{ display: "flex", gap: 8, flexWrap: "wrap" }}` 같은 인라인 스타일 사용. 통일 시 Tailwind로 전환:
```tsx
// Before
<form method="get" style={{ display: "flex", gap: 8, flexWrap: "wrap" }}>

// After
<form method="get" className="mb-6 flex flex-wrap gap-2">
```

버튼도 기존 HTML `<button>` 에서 `Button` 컴포넌트(`secondary` variant)로 교체.
