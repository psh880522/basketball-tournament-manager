  ├── 현재 모집 중인 대회 섹션
  │   └── <OpenTournamentsList> (Suspense)
  └── 현재 진행 중인 대회 섹션

```
<header> sticky top-0, h-14, bg-white/85 backdrop-blur
  ├── 로고: "🏀 23Board" (검정 텍스트)
  └── NavMenu (텍스트: slate-600/900)
```

### 현재 한계

- 히어로가 작고 평범함 (text-2xl 제목, 단순 amber 배경)
- 헤더가 흰색 — 브랜드 임팩트 없음

- 진행 상태(모집중/진행중/종료)를 색상으로 구분하나 카드 디자인이 동일함

### 5-1. 프론트엔드

#### 페이지 레이아웃 변경

**히어로 섹션** (HoopHub `<section class="relative min-h-[921px]">` 참고):
- 배경: 다크 (`#0a0e14` 계열) + 농구 코트 배경 이미지 + 그라디언트 오버레이

- 역할 분기 로직은 현행 유지

**대회 상태 섹션** (HoopHub `Real-Time Status` 참고):
- 그리드: `grid-cols-1 md:grid-cols-3`

**푸터**:

## 7. 구현 시 주의사항

### 7-1. 다크/라이트 혼합 배경 문제

현재 `app/globals.css`의 `body` 배경이 `#f0f0f0` (라이트). 랜딩 페이지만 다크로 만들려면:
- `body`에 배경색 제거 (투명화) 또는
- `page.tsx`에서 직접 `bg-[#0a0e14]` 배경 덮어쓰기

로그인 후 앱 영역(Sidebar 레이아웃)은 라이트 그대로 유지해야 하므로, **랜딩 페이지 `<main>`에만 다크 배경 적용**하는 방향이 안전.

[MEMO] 다크는 꼭 유지 하지 않아도 됨. 기존의 프로젝트에서 사용하던 색 사용해도 무방

### 7-2. 헤더 배경 전환

`GlobalHeader`는 비로그인 전용. 로그인 상태에서는 `Sidebar` 레이아웃으로 전환되므로, 헤더 다크화는 비로그인 경험에만 적용됨 → 안전.

### 7-3. NavMenu 텍스트 가시성

`NavMenu.tsx`는 현재 `text-slate-600/900` 고정값 사용. 다크 헤더에서 **가독성 심각하게 저하됨**. `className` prop이나 `theme` prop을 추가해 다크/라이트 모드를 분기하거나, 단순히 색상 직접 변경.

### 7-4. 배경 이미지 사용 여부

HoopHub의 히어로는 외부 이미지 URL 사용. 우리 프로젝트는:
- `public/` 폴더에 `logo.jpg`만 존재
- 히어로 배경 이미지로 농구 코트 사진이 필요 → `public/` 추가 필요
- 이미지 없이도 다크 그라디언트만으로 구현 가능 (MVP 선택지)

### 7-5. 하드코딩된 텍스트

현재 히어로 h1이 `"농심 3X3 농구대회에 오신 것을 환영합니다!"` 하드코딩. 개편 후에도 유지하되 타이포 스타일만 변경하거나, 더 범용적인 문구로 변경 여부 결정 필요.

### 7-6. 대회 카드 Suspense 유지

`<OpenTournamentsList>`, `<InProgressTournamentsList>` 컴포넌트는 Suspense로 감싸진 서버 컴포넌트. 레이아웃 변경 시에도 이 구조 유지 필수 (로딩 skeleton UI도 다크 테마에 맞게 수정).

## 9. 결론

### HoopHub에서 차용할 핵심 요소

| 요소 | 적용 방법 |
|------|-----------|
| 다크 히어로 (배경 이미지 + 오버레이) | `bg-[#0a0e14]` + 배경 이미지 레이어 (이미지 없으면 그라디언트 대체) |
| 초대형 이탤릭 헤드라인 | `font-space-grotesk text-7xl md:text-9xl font-black italic` |
| 글래스 모피즘 헤더 | `bg-[#0a0e14]/80 backdrop-blur-xl` |
| 상태별 컬러 보더 카드 | `border-l-4 border-{primary/secondary/tertiary}` |
| 오렌지 CTA 버튼 | `arena-gradient` (기존 primary 버튼과 동일) |
| 다크 푸터 | `bg-[#0a0e14] border-t border-white/10` |

### 프로젝트 특성상 제외할 요소

| HoopHub 요소 | 이유 |
|------|------|
| 스폰서 섹션 ("Elevate Your Brand") | 아마추어 토너먼트 매니저에 불필요 → 통계 섹션으로 대체 검토 |
| "Courtside Snapshot" 사진 갤러리 | 콘텐츠 없음 → 제외 |
| Live 스코어 카드 | 현재 실시간 스코어 기능 없음 → 진행 중 대회 카드로 대체 |
| "Watch Live" 버튼 | 스트리밍 기능 없음 → 제외 |

### 기대 효과

- 현재의 밋밋한 관리도구 인상 → 스포츠 브랜드 임팩트 강화
- 대회 상태를 색상 계층으로 직관적 표현
- Space Grotesk 폰트가 이미 로딩되어 있어 추가 로딩 비용 없음
- 역할 분기 비즈니스 로직은 전혀 건드리지 않아 회귀 위험 낮음
