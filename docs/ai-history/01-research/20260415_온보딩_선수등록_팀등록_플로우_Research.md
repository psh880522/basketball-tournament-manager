## 7. 구현 시 주의사항

### 7.1 DB 스키마 결정

**Issue**: teams.tournament_id가 필수인데, MVP에서는 팀이 tournament과 독립적으로 생성되어야 함.

**권장**: tournament_id를 nullable로 변경 (변경 최소화)

### 7.2 Role 타입 통일

**Issue**: team_members.role_in_team이 'manager'/'player'인데, MVP는 'captain'/'team_member' 필요.

**권장**: role_in_team 값 변경 + 마이그레이션

### 7.3 사용자 팀 상태 추적

**구현**: 런타임에 계산 (별도 테이블 불필요)
```typescript
async function getUserTeamStatus(userId: string): Promise<UserTeamStatus> {
  const teamMembers = await getTM(userId);
  if (teamMembers.length === 0) return "no_team";
  if (joinApplications.length > 0) return "join_pending";
  if (isCaptain) return "captain";
  return "team_member";
}
```

### 7.4 온보딩 "건너뛰기" 플로우

**구현**: 대시보드 배너로 미완료 항목 동적 표시

### 7.5 팀 생성 시 RPC 수정

현재:
```sql
INSERT INTO team_members (team_id, user_id, role_in_team)
VALUES (v_team_id, v_uid, 'manager');
```

수정 필요:
```sql
INSERT INTO team_members (team_id, user_id, role_in_team)
VALUES (v_team_id, v_uid, 'captain');
```

### 7.6 팀 합류 신청 플로우

**테이블**: team_join_applications (team_id, user_id, status='pending'|'approved'|'rejected')

**플로우**:
1. 사용자 신청 → team_join_applications insert
2. 팀장 승인 → status='approved' + team_members insert
3. 또는 거절 → status='rejected'

### 7.7 선수등록 완료 → 팀선택 전환점

**권장**: Option A (강제 플로우)
- 본인인증 완료 → 팀 선택 페이지로 즉시 이동
- `/onboarding/team-choice` 또는 `/teams/new-or-find`로 리다이렉트
