# 선수용 대시보드 UI 스펙

---

## 1. 리서치 파일 반영 요약

### 채택한 UI 패턴

- **요약 카드 상단 배치** — 스포츠 앱(ESPN, Ballzy)의 "나의 상황 한눈에" 패턴 차용. 숫자 기반 카드 5개로 현재 상태를 즉각적으로 스캔할 수 있게 구성.
- **액션 우선 메인 카드** — 통계 차트가 아니라 "지금 해야 할 일"을 가장 크게 노출. Google Workspace 대시보드의 Action Items 패턴과 동일한 철학.
- **팀 단위 묶음 표시** — 여러 팀 + 대회를 각각 분리하지 않고 팀 카드 아래에 대회 현황을 종속 표시 (Notion 사이드바 그룹 패턴). 다중 팀 멤버십 모델에 자연스럽게 대응.
- **경기 일정 통합 리스트** — 팀별로 분리하지 않고 날짜순 통합 뷰 제공 (Google Calendar 단일 뷰). 어느 팀의 경기인지는 팀명 배지로 표시.

### 의도적으로 피한 UI 패턴

- **차트/그래프 위주 레이아웃** (이유: 선수 대시보드는 의사결정 지원이 목적. 통계 분석은 관리자 영역. 차트 로딩은 인지 부하와 성능 비용 모두 증가.)
- **탭 기반 메인 네비게이션** (이유: 팀이 여러 개일 때 탭으로 분기하면 전체 맥락을 잃음. 스크롤로 모든 정보를 단일 피드에서 확인하는 것이 더 효율적.)
- **카드 내 인라인 편집 UI** (이유: 대시보드는 정보 허브. 편집은 해당 상세 페이지로 이동 유도. 대시보드의 인지 부하 최소화.)

### 타겟 사용자 특성이 UI에 영향을 준 결정

- **다중 팀 멤버십** → 모든 섹션에서 "어느 팀" 컨텍스트를 팀명 배지로 항상 명시
- **captain / player 혼재** → 액션 항목(입금, 로스터)은 captain인 팀에서만 노출. player 팀은 읽기 전용 뷰
- **모바일 주 사용 가능성** → 카드 레이아웃, 터치 친화적 최소 tap target(44px), DragScroll 수평 스크롤 활용
- **빠른 정보 스캔 요구** → 섹션 제목 + 배지 + 핵심 수치를 3초 내 파악 가능하도록 정보 계층 설계

### 타이포그래피

```
폰트 패밀리:
  - Heading: Space Grotesk (--font-space-grotesk)
  - Body:    Inter (--font-inter)

스케일 (globals.css 기준):
  display-lg  : 60px / Space Grotesk 700 / line-height 1.1  (랜딩 전용)
  headline-lg : 32px / Space Grotesk 700 / line-height 1.2  (페이지 제목)
  headline-md : 24px / Space Grotesk 600 / line-height 1.3  (섹션 제목)

Tailwind 클래스 기준:
  h1 (페이지 제목)  : text-2xl font-semibold   (24px)
  h2 (섹션 제목)   : text-lg font-semibold     (18px)
  h3 (카드 제목)   : text-sm font-medium       (14px)
  body            : text-sm                   (14px)
  caption         : text-xs text-[--ui-muted]  (12px)
```

### 간격 시스템

- 기본 단위: **4px** (Tailwind 기본 spacing 1 = 4px)
- 주요 spacing:
  - 카드 내부 패딩: `p-5` (20px)
  - 섹션 간격: `gap-6` (24px)
  - 카드 내 요소 간격: `gap-3` (12px) ~ `gap-4` (16px)
  - 상단 헤더 여백: `py-8 px-4` (32px / 16px)
  - 최대 너비: `max-w-5xl` (960px)

### 컴포넌트 스타일

**Card**
```
기본: bg-white rounded-xl p-5 shadow-sm
highlight: bg-[#FFF5EC] rounded-xl p-5     (브랜드 포인트 배경)
muted: bg-[#f0f0f0] rounded-xl p-5         (비활성)
```

**Badge**
```
default : bg-slate-100  text-slate-700     (기본 상태)
success : bg-emerald-100 text-emerald-700  (확정/완료)
warning : bg-amber-100  text-amber-800     (대기중)
danger  : bg-rose-100   text-rose-700      (취소/긴급)
info    : bg-sky-100    text-sky-700       (정보)
live    : bg-[#FF6B00]/10 text-[#FF6B00]   (캡틴/실시간)
```

**Button**
```
primary  : gradient #FF6B00→#FF8C33, white text, rounded-[4px]
secondary: border border-[#FF6B00]/30, text-[#FF6B00], transparent bg
ghost    : text-slate-600, hover:bg-surface-low, no border
```

**아이콘 라이브러리:** `lucide-react` 권장
- 현재 프로젝트에서 사용 여부 확인 후 없으면 `npm install lucide-react`
- 주요 아이콘: `Users`, `Trophy`, `Calendar`, `AlertCircle`, `ChevronRight`, `Clock`, `MapPin`, `Shield`

  ...
```

## 4. 핵심 화면 상세 설계

### 화면 A: 대시보드 — 팀 없음 (Empty State 1)

**목적:** 신규 선수가 팀을 구성하도록 유도

**레이아웃:**
```
┌─────────────────────────────────────┐
│ 헤더: "내 허브" + 사용자 이름        │
├─────────────────────────────────────┤
│                                     │
│     🏀 일러스트 (선택)               │
│                                     │
│  "아직 속한 팀이 없어요"              │
│  "팀을 만들거나 기존 팀에 합류하세요." │
│                                     │
│  [팀 만들기]   [팀 찾기]             │
│                                     │
├─────────────────────────────────────┤
│  둘러보기: 현재 모집 중인 대회 →      │
└─────────────────────────────────────┘
```

**주요 컴포넌트:**
- `Card variant="highlight"` — 온보딩 CTA 박스
- `Button variant="primary"` — 팀 만들기
- `Button variant="secondary"` — 팀 찾기
- 하단 대회 둘러보기 링크 카드 (기존 패턴 유지)

**사용자 액션:** `/teams/new`, `/teams/find` 이동

**엣지 케이스:**
- `join_pending` 상태: "신청 대기 중" amber 배너 표시 (팀 찾기 버튼 비활성)

**모바일 대응:** 동일 (이미 단일 컬럼 구조)

---

### 화면 B: 대시보드 — 팀 있음 + 활성 대회 없음 (Empty State 2)

**목적:** 팀은 있지만 대회 참가 전인 선수에게 대회 신청을 유도

**레이아웃:**
```
┌─────────────────────────────────────┐
│ 헤더: "내 허브" + 사용자 이름        │
├──────┬──────┬──────┬──────┬─────────┤
│ 팀 2 │만든1 │대회 0│경기 0│액션 0   │  ← 요약 카드
├─────────────────────────────────────┤
│  내 팀들                            │
│  ┌──────────────┐ ┌──────────────┐  │
│  │ 팀A   [팀장] │ │ 팀B   [팀원] │  │
│  │ 참가 대회 없음│ │ 참가 대회 없음│  │
│  │ [팀 보러가기]│ │ [팀 보러가기]│  │
│  └──────────────┘ └──────────────┘  │
├─────────────────────────────────────┤
│  🏆 현재 신청 가능한 대회 →          │
└─────────────────────────────────────┘
```

**주요 컴포넌트:**
- `SummaryCards` — 5종 요약 카드 (대회 0, 경기 0 표시)
- `MyTeamsGrid` — 팀 카드 (대회 없음 상태)
- 대회 둘러보기 유도 카드

**사용자 액션:** `/teams/[teamId]` 이동, `/tournaments` 이동

---

### 화면 C: 대시보드 — 풀 상태 (Full Dashboard) ★ 핵심

**목적:** 여러 팀에 속한 선수가 모든 활동을 한 화면에서 파악

**레이아웃 (데스크탑, 최대 너비 960px):**

```
┌─────────────────────────────────────────────────────┐
│ HEADER                                              │
│ "내 허브"            안녕하세요, [이름]님            │

├─────────────────────────────────────────────────────┤
│ 지금 해야 할 일    [2개]                             │  ← ActionItems (메인 카드)
│ ┌─────────────────────────────────────────────────┐ │
│ │ ⚠ [팀A] 코리아컵 입금 완료 표시 필요            │ │

│ │ 📋 [팀B] 서울리그 로스터 작성 필요              │ │

│ │ 코리아컵 · 조별리그 · 1코트 · 성동구체육관       ││
│ ├──────────────────────────────────────────────────┤│
│ │ 4/27(일) 10:00  |  [팀B] vs 스카이팀            ││
│ │ 서울리그 · 8강 · 3코트 · 마포체육관              ││
│ └──────────────────────────────────────────────────┘│
├─────────────────────────────────────────────────────┤
│ 내 팀들                                              │  ← MyTeamsGrid
│ ┌─────────────────────┐  ┌─────────────────────┐   │
│ │ 팀A  [팀장]          │  │ 팀B  [팀원]          │   │
│ │                     │  │                     │   │
│ │ 코리아컵            │  │ 서울리그             │   │

│ │                     │  │                     │   │

│ │ 4/25 14:00 vs 우리팀│  │ 4/27 10:00 vs 스카이│   │
│ │                     │  │                     │   │

├─────────────────────────────────────────────────────┤
│ 최근 결과                                            │  ← RecentResults
│ [팀A] 코리아컵 예선  승  72 - 58  vs 한강팀          │
│ [팀B] 서울리그 16강  패  45 - 61  vs 강남팀          │
└─────────────────────────────────────────────────────┘

**주요 컴포넌트 상세:**

#### SummaryCards (5개 카드)

```
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│  2   │ │  1   │ │  3   │ │  2   │ │  2   │
│소속팀│ │내가만│ │진행중│ │이번주│ │액션  │
│      │ │든 팀 │ │대회  │ │경기  │ │필요  │
└──────┘ └──────┘ └──────┘ └──────┘ └──────┘
```

- 5개 고정, 모바일에서 2×3 그리드 (마지막은 풀 너비 또는 2개 행)
- 액션 카드는 `> 0`일 때 `bg-[#FF6B00]/5` 강조
- 클릭 가능 카드 (→ 해당 섹션으로 스크롤 or 페이지 이동)

#### ActionItems (메인 카드, 가장 큰 카드)

```
prop: actions: PendingAction[]

액션 타입별 표시:
  payment          → 아이콘 ⚠ (amber), "입금 완료 표시 필요", 납입 기한 표시
  roster           → 아이콘 📋 (sky), "로스터 작성 필요", 현재 인원 표시
  team_join_approval → 아이콘 👥 (live), "팀원 신청 N건 승인 대기"

각 항목:
  - 팀명 배지 (badge live)
  - 액션 설명 텍스트
  - 부가 정보 (기한, 인원 등)
  - [바로가기] 버튼 (ghost, ChevronRight 아이콘)

빈 상태:
  - "지금 해야 할 일이 없어요 ✓" (success 색상)
```

#### UpcomingMatches

```
prop: matches: UpcomingMatch[]

각 경기 행:
  좌측: 날짜+시간 (caption, bold)
  중앙: [팀명 배지] vs 상대팀명, 대회명·단계, 장소/코트
  우측: 없음 (미래 경기이므로 스코어 없음)

구분선으로 날짜 그룹화 (선택사항):
  ── 4월 25일(금) ──

빈 상태: "예정된 경기가 없습니다"

최대 표시: 5건, "전체 보기 →" 링크 (향후 /schedule 페이지로)
```

#### MyTeamsGrid

```
prop: teams: MyTeamWithApplications[]

카드 구조:
  상단: 팀명 (h3) + 역할 배지 (captain→live, player→default)
  중단: 참가 중인 대회 목록 (각각 대회명 + 상태 배지 + 단계)
        대회 없으면 "참가 중인 대회 없음" (caption, muted)
  하단: 다음 경기 (있으면 날짜+상대팀, 없으면 빈칸)
        [팀 상세 보기] 버튼 (secondary)

captain 팀에만 추가:
  - 액션 배지 (입금 필요 등)
  - 팀원 신청 건수 표시

그리드: 데스크탑 md:grid-cols-2, 모바일 1열
```

#### RecentResults

```
prop: results: CompletedMatch[]

각 행:
  [팀명 배지] 대회명·단계  |  승/패  |  스코어  |  vs 상대팀

승: text-emerald-700 font-semibold "승"
패: text-rose-700    font-semibold "패"

최대 5건, 없으면 섹션 미노출
```

**사용자 액션:**
- 액션 항목 클릭 → `/my-applications/[id]` 이동
- 팀 상세 보기 → `/teams/[teamId]` 이동
- 경기 행 클릭 → 향후 경기 상세 페이지 (현재는 미구현, 비클릭 처리)
- "전체 보기" → 향후 `/schedule` 또는 현재 비활성

**엣지 케이스:**

| 상황 | 처리 |
|------|------|
| 액션 0건 | "할 일 없음" 성공 상태 카드 표시 |
| 경기 일정 없음 | UpcomingMatches 섹션 미노출 |
| 최근 결과 없음 | RecentResults 섹션 미노출 |
| 팀 1개 | 그리드 대신 풀 너비 단일 카드 |
| 팀이 매우 많음 (5+) | 기본 3개 + "더 보기" 토글 (초기 MVP에서는 스크롤로 처리) |
| API 에러 | 섹션별 에러 fallback (전체 페이지 crash 방지) |
| scheduled_at NULL | 경기 리스트에서 제외 (일정 미확정) |

**모바일 대응 (767px 이하):**
- SummaryCards: `grid-cols-2 gap-3` (5번째 카드 col-span-2)
- ActionItems: 동일 (이미 단일 컬럼)
- UpcomingMatches: 세로 리스트 (DragScroll 없이)
- MyTeamsGrid: `grid-cols-1`
- 헤더 서브텍스트 줄임 처리

### 우선 구현 순서 (의존성 순)

4. lib/api/applications.ts  [기존 수정]
   → listMyApplications() : captain 필터 제거
     (line 758: .eq("role_in_team", "captain") 삭제)

### 상태 관리가 필요한 화면

- `dashboard/page.tsx` 자체는 **Server Component** 유지 (force-dynamic)
- 클라이언트 상태 불필요 — 모든 데이터는 서버에서 조회
- 단, 향후 "더 보기" 토글이 필요하면 해당 컴포넌트만 `"use client"` 지정

### API 연동 필요 화면 목록

| API 함수 | 연동 위치 | 비고 |
|---------|---------|------|
| `getMyDashboardSummary()` | page.tsx | 신규 구현 필요 |
| `getMyPendingActions()` | page.tsx | 신규 구현 필요 |
| `listMyTeams()` | page.tsx | 기존 재사용 |
| `getMyUpcomingMatches()` | page.tsx | 신규 구현 필요 |
| `listMyApplications()` | page.tsx | 기존 수정 (captain 필터 제거) |
| `getMyRecentResults()` | page.tsx | 선택사항 |

**page.tsx 데이터 조합 패턴:**
```typescript
export const dynamic = "force-dynamic";
export const revalidate = 0;

export default async function DashboardPage() {
  const auth = await getUserWithRole();
  // 역할 리다이렉트 처리...

  const [summary, actions, teams, upcomingMatches, applications, recentResults] =
    await Promise.all([
      getMyDashboardSummary(),
      getMyPendingActions(),
      listMyTeams(),
      getMyUpcomingMatches(),
      listMyApplications(),
      getMyRecentResults(),
    ]);

  const hasTeams = (teams.data ?? []).length > 0;
  const hasActiveApps = (applications.data ?? []).some(
    (a) => a.status === "confirmed"
  );

  if (!hasTeams) return <EmptyDashboard state="no_team" />;
  if (!hasActiveApps) return <EmptyDashboard state="no_app" teams={teams.data} />;

  return (
    <main className="min-h-screen bg-page px-4 py-8">
      <div className="mx-auto flex max-w-5xl flex-col gap-6">
        <header>...</header>
        <SummaryCards summary={summary.data} />
        <ActionItems actions={actions.data ?? []} />
        <UpcomingMatches matches={upcomingMatches.data ?? []} />
        <MyTeamsGrid teams={...} applications={applications.data ?? []} />
        <RecentResults results={recentResults.data ?? []} />
      </div>
    </main>
  );
}
```
