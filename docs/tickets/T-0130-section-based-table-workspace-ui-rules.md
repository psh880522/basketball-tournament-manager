# Vertical Slice Ticket

## 목표

서비스 전체 UI 기준(T-0128, T-0129)을 보완하여,
경기 리스트 계열 화면에서는 **카드 중첩을 줄이고 섹션 + 테이블 구조**를 기본으로 사용하도록 통일한다.

이번 슬라이스 완료 시 다음이 가능해야 한다.

- 경기목록 / 스케줄  / 결과입력 화면의 기능 구분이 카드가 아니라 섹션으로 정리됨
- 코트 카드 > division 카드 > 리그/토너먼트 섹션 > 테이블 구조가 공통 규칙으로 확정됨
- 테이블에 `구분` 컬럼이 추가되어 조 이름 / 라운드 이름을 별도로 표시함
- 경기 컬럼에는 순수 경기명만 표시하도록 규칙이 통일됨

DB 스키마 변경 없음

신규 테이블 / 신규 컬럼 / migration 생성 금지

## API

신규 비즈니스 API 추가 없음

허용 범위

- 테이블 표시용 formatter / mapper / helper 추가 가능
- `구분` 컬럼 표시용 view helper 추가 가능
- 섹션 표시용 view model helper 추가 가능

예시

- league section label formatter
- tournament round formatter

- match display text formatter

## UI

경기 리스트 계열 화면의 공통 구조를 아래로 통일한다.

### 공통 상위 구조

```text
코트(카드)
  division(카드)
    리그(섹션)
      [테이블]

    토너먼트(섹션)
      [테이블]
```

즉

- 코트 = 카드
- division = 카드
- 기능 구분 = 섹션
- 실제 반복 데이터 = 테이블

기능별 카드 중첩은 사용하지 않는다.

---

### 섹션 구조 원칙

다음과 같은 화면에서는 기능을 카드로 나누지 않고 섹션으로 구분한다.

대상 예시

- result 페이지
- schedule 페이지의 경기 리스트 영역
- match list 페이지

예시

```text
[페이지 헤더]

=== 리그 결과 입력 ===
[테이블]

=== 리그 순위 계산 ===
[버튼]
[순위표]

=== 토너먼트 팀 배치 ===
[미리보기]
[버튼]

=== 토너먼트 결과 입력 ===
[테이블]
```

원칙

- 기능별 묶음은 섹션 타이틀로 나눈다
- 테이블은 섹션 바로 아래에 배치한다
- 카드 안에 또 카드 안에 테이블 구조는 피한다

---

### 경기 리스트 / 스케줄 / 결과입력 공통 테이블 규칙

테이블에 `구분` 컬럼을 공통으로 추가한다.

`구분` 컬럼 역할

- 리그: A조 / B조 / C조
- 토너먼트: 16강 / 8강 / 4강 / 결승
- 휴식시간: 휴식시간
- 필요 시 maintenance / buffer도 표시 가능

경기 컬럼 규칙

- 경기 컬럼에서는 순수 경기명만 표시
- 기존 `[A조]`, `[4강]` 같은 접두 표시는 경기 컬럼에서 제거
- 조/강 정보는 `구분` 컬럼으로만 표시

---

### 1. 경기목록 표시 규칙

구조

```text
코트(카드)
  division(카드)
    리그(섹션)
      [테이블]
      시간 / 구분 / 경기 / 상태 / 점수

    토너먼트(섹션)
      [테이블]
      시간 / 구분 / 경기 / 상태 / 점수
```

예시

리그

- 시간: 09:00
- 구분: A조
- 경기: Tigers vs Hawks
- 상태: 예정
- 점수: -

토너먼트

- 시간: 11:00
- 구분: 4강
- 경기: 1위팀 vs 4위팀
- 상태: 예정
- 점수: -

---

### 2. 스케줄  표시 규칙

구조

```text
코트(카드)
  division(카드)
    리그(섹션)
      [테이블]
      시간 / 구분 / 경기(드래그앤드롭) / 코트(드롭다운)

    토너먼트(섹션)
      [테이블]
      시간 / 구분 / 경기(드래그앤드롭) / 코트(드롭다운)
```

컬럼

- 시간
- 구분
- 경기
- 코트

규칙

- 시간은 slot 기준으로 고정
- DnD는 경기 교체/배치 기준으로 해석
- 구분 컬럼은 읽기 전용
- 휴식시간은 별도 섹션이 아니라 같은 테이블 행으로 표시

휴식시간 예시

- 시간: 10:00 - 10:10
- 구분: 휴식시간
- 경기: -
- 코트: 코트1

---

### 3. 결과입력 표시 규칙

구조

```text
코트(카드)
  division(카드)
    리그(섹션)
      [테이블]
      시간 / 구분 / 경기 / 점수입력 / 상태(배지) / 저장(버튼)

    토너먼트(섹션)
      [테이블]
      시간 / 구분 / 경기 / 코트 / 점수입력 / 상태(배지) / 저장(버튼)
```

리그 컬럼

- 시간
- 구분
- 경기
- 점수
- 상태
- 저장

토너먼트 컬럼

- 시간
- 구분
- 경기
- 코트
- 점수
- 상태
- 저장

---

### 경기명 표시 규칙

리그

- `Tigers vs Hawks`

토너먼트

- `1위팀 vs 4위팀`
- 또는 `Tigers vs Hawks`

휴식시간

- 경기 컬럼은 `-` 또는 비움

즉 조/강 이름은 경기 컬럼에 포함하지 않는다.

---

### 상태 표시 규칙

공통 배지 예시

- 예정
- 진행중
- 완료
- 미배정
- 주의 필요

상태 배지는 경기목록 / 스케줄 / 결과입력 화면에서 동일한 스타일과 이름을 사용한다.

---

### 적용 대상 페이지

최소 아래 페이지들에 적용 가능해야 한다.

- /admin/tournaments/[id]/matches
- /admin/tournaments/[id]/schedule
- /admin/tournaments/[id]/result

필요 시 standings / 팀/선수 리스트와는 다른 “경기 리스트 계열 전용 표시 규칙”으로 유지한다.

## 완료 기준

- 경기 리스트 계열 화면의 공통 구조가 “코트 카드 > division 카드 > 섹션 > 테이블”로 정리됨
- 기능별 카드 중첩 대신 섹션 구조를 사용한다는 기준이 확정됨
- 경기목록 / 스케줄  / 결과입력 테이블에 `구분` 컬럼이 포함됨
- 조 이름 / 강 이름은 `구분` 컬럼에만 표시됨
- 경기 컬럼에서는 `[조]`, `[강]` 접두 표기가 제거됨
- 이후 schedule / result / matches 화면이 같은 테이블형 기준으로 구현 가능해짐
- 기존 기능 동작에는 영향 없음
