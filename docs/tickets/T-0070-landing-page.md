# Vertical Slice Ticket

## 목표

- 서비스 첫 화면(`/`)에서
  - 이 서비스가 무엇인지 명확히 전달한다
  - 현재 참여 가능한 대회를 즉시 보여준다
  - 다음 행동(참가 / 운영)이 명확하게 드러난다
- 기존 Tournament List 기능을 재사용하면서 UX를 개선한다

---

## 범위 요약 (중요)

- 이번 슬라이스는 UI 구성과 상태 표시 개선이 핵심이다
- 새로운 도메인 로직은 추가하지 않는다
- DB 구조 변경은 없다
- 마케팅용 디자인 고도화는 포함하지 않는다

---

## 랜딩 페이지 구성

### 경로
`/app/page.tsx`

---

### 1️⃣ Hero 영역

- 서비스 한 줄 설명
  - 예: “조별 리그 기반 농구대회를 한 번에 관리하세요”
- 보조 설명 1~2줄
- CTA 버튼 2개
  - “대회 참가하기”
  - “대회 운영하기”

#### 버튼 동작

- 로그인 안 한 경우
  - 참가하기 → `/login`
  - 운영하기 → `/login`
- 로그인 한 경우
  - 참가하기 → `/tournaments` 또는 open 대회 목록
  - 운영하기 → `/admin`

---

### 2️⃣ 현재 모집 중 대회 섹션

- `tournament.status = open` 인 대회만 표시
- 없을 경우:
  - “현재 모집 중인 대회가 없습니다”

표시 정보:
- 대회명
- 일정
- 장소
- 모집 상태 (모집 중)

---

### 3️⃣ 기타 상태 대회 (선택)

- `closed`, `finished` 대회는 하단에 간단히 표시하거나 생략 가능
- 이번 티켓에서는 open 강조가 핵심

---

## 데이터 로딩 방식

- Server Component에서 Supabase 조회
- 최소 조회 필드:
  - id
  - name
  - location
  - start_date
  - end_date
  - status

- open 상태 우선 정렬

---

## UI 상태 처리

- 로딩 상태
- open 대회 없음 상태
- 에러 발생 시 기본 안내 메시지

---

## 권한

- 비로그인 사용자 접근 가능
- 로그인 사용자도 동일하게 접근 가능

## 수정 허용 범위 (필수)
