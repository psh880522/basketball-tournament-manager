# Vertical Slice Ticket

## 목표

경기 리스트 계열 화면의 테이블 표시 규칙을 보강하여,  
토너먼트 경기 표기와 컬럼 폭 동작을 서비스 전체에서 일관되게 만든다.

이번 슬라이스 완료 시 다음이 가능해야 한다.

- 토너먼트 테이블의 `구분` 컬럼에 라운드 + 경기 번호가 표시됨
- 결승 / 3·4위전은 이전 경기 참조형 경기명으로 표시 가능
- 실제 팀이 배치되면 참조형 표기 대신 실제 팀명으로 자동 전환됨
- 시간 / 구분 / 상태 / 점수 / 저장 / 코트 컬럼은 고정폭으로 동작함
- 화면 폭이 변해도 경기 컬럼만 유동적으로 줄고 늘어남
- result 페이지 등 테이블 중심 화면은 카드 중첩 없이 섹션 + 테이블 구조를 유지함

---

## DB

DB 스키마 변경 없음

이번 슬라이스는 UI 표시 규칙과 테이블 레이아웃 기준만 정리한다.

신규 테이블 / 신규 컬럼 / migration 생성 금지

---

## API

신규 비즈니스 API 추가 없음

허용 범위

- 토너먼트 경기 표시용 formatter / helper 추가 가능
- `구분` 컬럼 표기용 formatter 추가 가능
- 참조형 경기명 formatter 추가 가능
- 테이블 컬럼 정의 helper 추가 가능

예시

- tournamentRoundLabel(match)
- tournamentMatchLabel(match)
- sectionColumnConfig(type)
- matchDisplayText(match)

## UI

경기 리스트 계열 테이블 표시 규칙을 아래로 보강한다.

### 1. 테이블 상위 구조

경기 리스트 계열 화면은 아래 구조를 유지한다.

```text
코트(카드)
  division(카드)
    리그(섹션)
      [테이블]

    토너먼트(섹션)
      [테이블]
```

기능 구분은 카드가 아니라 섹션으로 유지한다.

적용 대상 예시

- /admin/tournaments/[id]/matches
- /admin/tournaments/[id]/schedule
- /admin/tournaments/[id]/result

---

### 2. `구분` 컬럼 표기 규칙

`구분` 컬럼은 경기 단계 식별용으로 사용한다.

#### 리그

예시

- A조
- B조
- C조

#### 토너먼트

같은 라운드 안의 경기 번호까지 포함해서 표시한다.

예시

- 16강 1경기
- 16강 2경기
- 8강 1경기
- 8강 2경기
- 4강 1경기
- 4강 2경기
- 결승
- 3/4위전

#### 휴식시간 / 운영 슬롯

예시

- 휴식시간
- 버퍼
- 정비

---

### 3. 경기 컬럼 표시 규칙

경기 컬럼에서는 조/강 접두어를 제거하고 순수 경기명만 표시한다.

#### 리그

예시

- Tigers vs Hawks

#### 토너먼트 초기 라운드

시드 기반으로 표시

예시

- 1위 vs 4위
- 2위 vs 3위

#### 결승 / 3·4위전

이전 경기 참조형으로 표시

예시

- 1경기 승자 vs 2경기 승자
- 1경기 패자 vs 2경기 패자

#### 실제 팀 확정 후

실제 팀명이 우선

예시

- Tigers vs Hawks
- Bears vs Eagles

표시 우선순위

1. 실제 팀이 배치되어 있으면 실제 팀명 표시
2. 실제 팀이 없고 이전 경기 참조 구조가 있으면 참조형 표시
3. 초기 시드 경기면 시드 기반 표시

휴식시간 / 버퍼 / 정비 슬롯은 경기 컬럼을 `-` 또는 비움으로 표시 가능

### 7. 컬럼 폭 규칙

테이블 컬럼 폭은 아래 원칙을 사용한다.

#### 고정폭 컬럼

- 시간
- 구분
- 상태
- 점수
- 저장
- 코트

이 컬럼들은 내용 길이가 예측 가능하므로 고정폭을 사용한다.

예시 가이드

- 시간: 110~140px
- 구분: 90~120px
- 상태: 80~100px
- 점수: 90~110px
- 저장: 80~100px
- 코트: 100~120px

#### 가변폭 컬럼

- 경기

경기 컬럼은 남는 공간을 우선 사용한다.

화면 크기 변경 시

- 시간 / 구분 / 상태 / 점수 / 저장 / 코트 컬럼은 유지
- 경기 컬럼만 유동적으로 줄고 늘어난다

긴 경기명 처리

- 한 줄 우선
- 길면 말줄임표 처리
- hover 또는 title로 전체 경기명 확인 가능

---

### 8. 테이블 레이아웃 원칙

권장 방식

- `colgroup` 또는 동등한 컬럼 폭 정의 방식 사용
- 고정폭 컬럼과 가변폭 경기 컬럼을 명시적으로 분리
- 경기 리스트 계열 테이블은 가급적 동일한 레이아웃 시스템 사용

## 권한

이번 슬라이스는 권한 로직 변경이 목적이 아니다.

기존 권한 정책을 유지하되, UI 표시 구조와 테이블 규칙만 통일한다.

- organizer: 기존 편집 UI 유지
- team_manager / player / viewer: 기존 조회 정책 유지

## 완료 기준

- 토너먼트 테이블의 `구분` 컬럼에 경기 번호 포함 표기가 가능함
- 결승 / 3·4위전에서 참조형 경기명 표시가 가능함
- 실제 팀 배치 후 참조형 경기명이 실제 팀명으로 전환됨
- 경기 리스트 계열 테이블에 고정폭 컬럼 / 가변폭 경기 컬럼 규칙이 정리됨
- 화면 폭이 변경되어도 경기 컬럼만 유동적으로 동작함
- result 등 테이블 중심 화면에서 카드 중첩 대신 섹션 + 테이블 구조를 유지함
- 기존 기능 동작에는 영향 없음
