# Vertical Slice Ticket

## 목표

서비스 전체 UI를 "스포츠 운영 서비스다운 느낌"으로 통일하기 위한 공통 UI 기준과 레이아웃 기반을 만든다.

이번 슬라이스에서는 기능 추가보다 **서비스 전반의 시각/구조 일관성 확보**를 목표로 한다.

이 슬라이스 완료 시 다음이 가능해야 한다.

- 공개 페이지 / 내부 페이지 / 관리자 페이지가 공통된 UI 규칙을 따름
- 버튼 / 카드 / 배지 / 폼 / 상태 표현 규칙이 공통화됨
- 경기 / 팀 / 선수 / 순위 / 슬롯 표시 규칙이 통일됨
- 이후 운영홈, bracket, schedule, result, standings에 같은 기준을 적용할 수 있음

---

## DB

DB 스키마 변경 없음

이번 슬라이스에서는 UI 시스템과 레이아웃 기준만 정리한다.

신규 테이블 / 신규 컬럼 / migration 생성 금지

---

## API

신규 비즈니스 API 추가 없음

허용 범위

- UI 렌더링에 필요한 최소 타입 정리
- 공통 표시용 helper / formatter 추가 가능
- 상태 배지, 도메인 표시 규칙용 view helper 추가 가능

예시

- 경기 상태 표시 formatter
- 단계 상태 badge mapper
- 슬롯 표시용 label helper
- 날짜/시간 포맷 helper

---

## UI

서비스 전체에 공통 적용할 UI 기준을 도입한다.

### UI 톤

- 스포츠 운영 서비스다운 느낌
- 깔끔한 SaaS 운영툴 + 스포츠 대회 보드 감성
- 빠르게 읽히고 상태 구분이 명확해야 함
- 예쁜 것보다 운영 정보 전달과 현장 사용성을 우선

---

### 공통 페이지 구조

모든 주요 페이지는 가능하면 아래 구조를 따른다.

1. 페이지 헤더
   - 제목
   - 설명 1줄
   - 주요 액션 버튼

2. 요약 영역
   - 상태 카드 / 숫자 카드 / 다음 해야 할 일

3. 주 작업 영역
   - 폼 / 리스트 / 보드 / 표

4. 보조 영역
   - 안내 / 주의 필요 항목 / 최근 상태

---

### 레이아웃 분류

#### 공개 페이지
대회 소개 / 참가 / 조회 중심

특징
- 넓은 카드
- 큰 제목
- CTA 명확
- 정보 밀도 낮음

대상 예시
- 랜딩
- 대회 목록
- 대회 상세
- 참가 신청

#### 앱 내부 페이지
팀 매니저 / 일반 사용자 작업 중심

특징
- 상단 타이틀 + 설명 + 액션
- 카드/섹션 기반
- 정보 밀도 중간

대상 예시
- 팀 대시보드
- 선수 등록
- 팀 관리

#### 관리자 페이지
운영 콘솔 중심

특징
- 현황 카드 + 작업 카드 + 운영 보드 구조
- 상태 배지 강조
- 정보 밀도 높음
- 빠른 판단 가능해야 함

대상 예시
- 운영 홈
- bracket
- schedule
- result
- standings

---

### 공통 컴포넌트 규칙

#### 버튼

Primary
- 현재 페이지의 핵심 액션 1개만 강조

Secondary
- 보조 액션

Danger
- 초기화 / 삭제 / 되돌리기

#### 카드

요약 카드
- 숫자 / 상태 중심

작업 카드
- 입력폼 + 실행 버튼

리스트 카드
- 팀 / 선수 / 경기 등 1건 표시

슬롯 카드
- 시간 / 코트 / 내용 표시

#### 배지

공통 상태 배지 예시

- 완료
- 진행중
- 미시작
- 대기
- 주의 필요
- 공개
- 비공개

배지명 / 색상 / 스타일은 전 서비스에서 공통 사용

#### 폼

원칙
- label 항상 표시
- 단위 표시 (`분`, `명`, `강`)
- 설명 문구는 짧게
- 관련 입력은 같은 카드 안에 묶음

---

### 도메인 표시 규칙

#### 경기 카드

항상 아래 순서로 표시

1. 시간
2. 코트
3. 경기 내용

예시

- 09:00 - 09:30
- 코트 1
- [A조] Tigers vs Hawks

토너먼트 예시

- 11:00 - 11:30
- 코트 2
- [토너먼트] 준결승 1

휴식시간 예시

- 10:00 - 10:10
- 코트 1
- [휴식시간]

#### 팀 카드

표시 순서

1. 팀명
2. 디비전
3. 상태
4. 선수 수

#### 선수 카드 / 리스트

표시 순서

1. 이름
2. 등번호
3. 포지션
4. 소속 팀

#### 순위표

우선 컬럼

- 순위
- 팀명
- 승
- 패
- 득점
- 실점

#### 슬롯 보드

구조 원칙

- 코트 > 디비전 > 조 / 토너먼트
- 시간은 읽기 쉽게 표시
- 휴식시간도 일반 슬롯처럼 보임
- 운영 보드는 보드형 카드 구조 유지

---

### 페이지별 적용 기준

#### 운영 홈
- 현황판 + 단계 카드
- 다음 해야 할 일 강조

#### bracket
- 생성 콘솔 + 구조 확인

#### schedule
- 생성 패널 + 운영 보드

#### result
- 진행 제어 콘솔

#### standings
- 공식 기록 / 순위표 중심

---

### 시각 스타일 방향

- 배경은 밝고 깔끔하게
- 카드 기반 흰색 영역
- 강조색 1개를 스포츠 서비스 포인트 컬러로 사용
- 상태색은 완료 / 진행중 / 경고만 명확히 구분
- 둥근 모서리와 그림자는 과하지 않게 사용

---

## 권한

이번 슬라이스는 권한 로직 변경이 목적이 아니다.

모든 역할에서 공통 UI 표현 기준은 동일하게 적용하되,
기존 권한 정책에 따라 표시/비활성/숨김 여부만 유지한다.

- organizer: 편집/실행 UI 노출
- team_manager / player / viewer: 기존 정책 유지
