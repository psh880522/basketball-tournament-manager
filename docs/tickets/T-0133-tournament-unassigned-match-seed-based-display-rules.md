# Vertical Slice Ticket

## 목표

토너먼트 미배정 경기의 표시 형식을 토너먼트 규칙에 맞는 시드/참조형 표기로 통일한다.

이번 슬라이스에서는 토너먼트 경기의 팀이 아직 실제 배정되지 않았더라도  
경기 컬럼에서 운영자가 의미를 바로 이해할 수 있도록 표시 규칙을 정리한다.

이 슬라이스 완료 시 다음이 가능해야 한다.

- 초기 토너먼트 라운드 미배정 경기는 시드 기준으로 표시됨
- 결승 / 3·4위전 등 후속 라운드 미배정 경기는 이전 경기 참조형으로 표시됨
- 실제 팀 배정 후에는 실제 팀명으로 자연스럽게 전환됨
- 경기목록 / 스케줄 슬롯 / 결과입력 화면에서 동일한 규칙을 사용함

---

## DB

DB 스키마 변경 없음

신규 테이블 / 신규 컬럼 / migration 생성 금지

이번 슬라이스는 표시 규칙만 수정한다.

---

## API

신규 비즈니스 API 추가 없음

허용 범위

- 토너먼트 경기 표시용 formatter / helper 추가 가능
- 시드 기반 표기 formatter 추가 가능
- 이전 경기 참조형 표기 formatter 추가 가능
- 경기 표시 우선순위 로직 보완 가능

표시 우선순위

1. 실제 팀이 모두 배정되어 있으면 실제 팀명 표시
2. 실제 팀이 없고 초기 라운드 시드 정보가 있으면 시드 기반 표시
3. 실제 팀이 없고 이전 경기 참조 정보가 있으면 참조형 표시

## UI

토너먼트 경기 표시 형식을 아래 규칙으로 통일한다.

### 적용 대상

- /admin/tournaments/[id]/matches
- /admin/tournaments/[id]/schedule
- /admin/tournaments/[id]/result

---

### 구분 컬럼 규칙

구분 컬럼은 기존 규칙 유지

예시

- 16강 1경기
- 8강 2경기
- 4강 1경기
- 결승
- 3/4위전

---

### 경기 컬럼 규칙

#### 1. 실제 팀 배정 완료
예시

- Tigers vs Hawks
- Bears vs Eagles

#### 2. 초기 라운드 미배정 경기
시드 규칙으로 표시

예시

- 1위 vs 4위
- 2위 vs 3위
- 1위 vs 8위
- 4위 vs 5위

#### 3. 후속 라운드 미배정 경기
이전 경기 참조형으로 표시

예시

- 1경기 승자 vs 2경기 승자
- 3경기 승자 vs 4경기 승자
- 1경기 패자 vs 2경기 패자

---

### 결승 / 3·4위전 규칙

결승

- `1경기 승자 vs 2경기 승자`

3/4위전

- `1경기 패자 vs 2경기 패자`

실제 팀이 배정되면

- Tigers vs Hawks
- Bears vs Eagles

로 전환한다.

---

### 금지 규칙

다음 표기는 사용하지 않는다.

- 미배정 경기
- 빈 경기
- TBD vs TBD
- - vs -

---

### 화면별 규칙

#### 경기목록

- 경기 컬럼: 시드 기반 또는 참조형 표시
- 상태 컬럼: 예정 / 미배정 / 진행중 / 완료 기존 규칙 유지

#### 스케줄 슬롯

- 경기 컬럼: 시드 기반 또는 참조형 표시
- 구분 컬럼: 라운드/경기 번호 유지
- 코트 컬럼은 기존 규칙 유지

#### 결과입력

- 경기 컬럼: 시드 기반 또는 참조형 표시
- 실제 팀 미배정 상태에서는 점수 입력 비활성 또는 차단 가능
- 실제 팀 배정 후 점수 입력 가능

---

## 권한

이번 슬라이스는 권한 로직 변경이 목적이 아니다.

기존 권한 정책 유지

- organizer: 기존 편집/실행 규칙 유지
- team_manager / player / viewer: 기존 조회 규칙 유지
