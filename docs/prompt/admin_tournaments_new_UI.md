# 작업 목표: '대회 생성 페이지' UI 개선 및 구현을 위한 기초 리서치

지금부터 admin/tournaments/new 페이지 UI를 개선 및 구현을 할거야.
기능 구현하다보니 먼저 적용된 기능들이 있어 admin/tournaments/[id]/edit 페이지도 리서치 범위에 넣어줘

너의 첫 번째 작업은 현재 프로젝트의 전체적인 폴더 구조와 핵심 파일들을 깊이 읽고 이해하는 거야. 이 정보를 바탕으로 앞으로 우리가 어떻게 UI를 수정하는게 좋을지 계획을 세워야해.

**수행할 작업:**
1. 프로젝트 루트 폴더를 전체적으로 깊이 읽고, 언어, 프레임워크, 라이브러리 등 주요 기술 스택을 파악해.
2. 각 폴더와 파일의 역할을 상세히 이해해.
3. 발견한 모든 세부 사항을 담은 상세 보고서를 `ai_docs/Research.md` 파일에 마크다운 형식으로 작성해.

**주의 사항:**
- 채팅창에서 구두로 요약하지 마. 반드시 파일에 작성해.
- '깊이', '매우 상세히', '세부 사항 파악' 등 너의 언어 모델적 특성을 살려 철저하게 분석해.



# 작업 목표: '대회 생성 페이지' UI 개선 및 구현을 위한 계획 수립

`ai_docs/Research.md` 문서를 잘 검토했어. 이제 이 정보를 바탕으로 대회 생성 페이지' UI 개선 및 구현을 위한 계획을 세워줘.

**수행할 작업:**
1. 구현해야 할 기능에 대한 상세 설명.
2. 어떤 라이브러리(예: `jsonwebtoken`, `bcrypt`)를 추가로 설치해야 하는지 기술.
3. 실제 변경될 파일 경로 및 추가될 파일 목록 (예: `src/models/User.js`, `src/routes/auth.js`, `src/controllers/auth.controller.js`).
4. 각 파일에 구현될 코드 스니펫(Pseudocode 또는 핵심 로직 포함).
5. 발생 가능한 트레이드 오프 또는 고려 사항.
6. 이 모든 내용을 담은 상세 플랜을 `ai_docs/Plan.md` 파일에 한글로 작성해.

**주의 사항:**
- **절대 코드를 바로 구현하지 마.** 나는 너의 계획만 먼저 검토할 거야.
- 구현을 시작하기 전에 내 승인을 받아야 해.

# 작업 목표: '대회 생성 페이지' UI 개선 및 구현을 위한 실제 코드 구현

최종 승인된 계획인 `ai_docs/Plan.md` 문서를 기반으로 실제 코드를 구현할 시간이야.

**수행할 작업:**
1. 계획에 명시된 모든 필요 라이브러리를 설치해.

3. 작업이나 단계를 완료할 때마다 `ai_docs/Plan.md` 문서에서 해당 섹션을 '[완료]'로 표시해.
4. 모든 작업이 완료될 때까지 멈추지 말고 기계적으로 쭉 실행해.
5. 타입을 사용하는 경우(예: TypeScript), 작업 중간중간 타입 체크를 실행하여 새로운 문제를 만들지 마.

**주의 사항:**
