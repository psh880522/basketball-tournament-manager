# 팀 단계 플로우 정리
프로필 등록 완료 이후: 팀 생성 / 팀 합류

## 1. 전체 흐름 내 위치

회원가입 → 선수 등록 → 본인인증 → 프로필 등록 완료 → 팀 생성/합류 → 대회 참가

여기서 `팀 생성/합류` 는
프로필 등록 완료 후 진입하는 하나의 큰 단계이며,
내부적으로는 아래 두 갈래로 나뉜다.

- 팀 생성 플로우
- 팀 합류 플로우

---

## 2. 팀 단계 진입 방식

### 2-1. 프로필 등록 완료 화면
프로필 등록 완료 후 다음 CTA 제공

버튼:
- [건너뛰기]
- [팀 생성/합류]

또는 확장형:
- [건너뛰기]
- [팀 만들기]
- [팀 찾기]

MVP에서는 우선 `[팀 생성/합류]` 버튼 하나로 진입시키고,
다음 화면에서 생성/합류를 선택하게 해도 된다.

### 2-2. 재진입 경로
건너뛰기 이후에도 아래 경로로 재진입 가능

사이드 메뉴:
- 팀 생성
- 팀 찾기

대시보드 상단 배너:
- "팀을 만들거나 기존 팀에 합류해보세요"

---

## 3. 팀 관련 사용자 상태

### 3-1. no_team
- 팀이 없는 상태
- 팀 생성 가능
- 팀 합류 신청 가능

### 3-2. join_pending
- 특정 팀에 합류 신청 후 승인 대기 상태
- 다른 팀 중복 신청 정책은 MVP에서 제한 가능
- 승인 전까지 대회 참가 불가

### 3-3. team_member
- 팀 소속이 확정된 상태
- 팀 정보 열람 가능
- 팀원으로 활동 가능

### 3-4. captain
- 팀 생성자 또는 팀장 권한 보유자
- 팀 관리 가능
- 팀원 승인/거절 가능
- 대회 참가 신청 가능

---

## 4. 팀 생성 플로우

### 4-1. 진입 조건
- 선수 등록 + 본인인증 완료 사용자
- 현재 팀 미소속 상태

### 4-2. 입력 항목
MVP 기준 최소 항목:
- 팀명
- 활동 지역
- 팀 소개

선택 항목:
- 팀 로고/대표 이미지

### 4-3. 생성 처리
팀 생성 완료 시 아래를 동시에 처리

- team 생성
- 생성자 = captain 지정
- 생성자 = 첫 번째 active team_member 자동 등록

즉,
팀 생성자는 별도 합류 절차 없이
자동으로 해당 팀의 팀장 겸 첫 팀원이 된다.

### 4-4. 생성 완료 후 이동
생성 완료 후 팀 상세/관리 화면으로 이동

노출 액션 예시:
- 팀 정보 수정
- 팀원 초대/승인 관리
- 팀 찾기 대신 팀 관리 중심 UI 노출
- 대회 참가 CTA 노출

---

## 5. 팀 합류 플로우

### 5-1. 진입 조건
- 선수 등록 + 본인인증 완료 사용자
- 현재 팀 미소속 상태

### 5-2. 팀 찾기 화면
사용자는 팀 목록/검색 화면에서 원하는 팀을 찾는다.

탐색 요소 예시:
- 팀명 검색
- 지역 필터
- 팀 소개 미리보기

### 5-3. 합류 신청
사용자는 팀 상세에서 [합류 신청] 버튼을 눌러 신청한다.

신청 후 상태:
- join_pending

### 5-4. 팀장 승인
해당 팀 captain은 신청자를 확인하고
- 승인
- 거절
처리할 수 있다.

### 5-5. 승인 완료
승인되면 상태가
- team_member
로 전환된다.

이후 사용자는 팀 상세/내 팀 화면으로 이동한다.

---

## 6. 팀장 관점 관리 플로우

### 6-1. 신청자 확인
팀장은 팀 관리 화면에서
합류 신청 목록을 확인한다.

### 6-2. 처리 액션
각 신청자에 대해
- 승인
- 거절
가능

### 6-3. 승인 결과
승인 시:
- 해당 사용자를 team_member active 상태로 등록

거절 시:
- 요청 종료
- 필요 시 재신청 허용 가능

## 7. UX 분기 규칙

### 7-1. 팀이 없는 사용자
노출:
- 팀 생성
- 팀 찾기
- 대시보드 배너: 팀 단계 유도

### 7-2. 합류 승인 대기 사용자
노출:
- "팀 승인 대기 중" 상태 표시
- 중복 행동 제한
- 필요 시 신청 취소 기능 검토

### 7-3. 팀 소속 사용자
노출:
- 내 팀
- 팀 정보
- 대회 참가
비노출 또는 축소:
- 팀 생성
- 팀 찾기

### 7-4. 팀장 사용자
추가 노출:
- 팀 관리
- 합류 요청 관리
- 대회 참가 신청

---

## 8. 권장 정책

### 8-1. 동시 소속 제한
MVP에서는 사용자 1명당 동시 1팀만 허용

### 8-2. 중복 신청 제한
MVP에서는 한 번에 1개 팀만 합류 신청 가능하게 제한하는 것이 단순함

### 8-3. 팀 생성 후 합류
팀 생성자는 자동 팀 소속 처리되므로
별도 합류 절차 없음

### 8-4. 팀 미소속 상태에서만 팀 생성/합류 가능
이미 팀이 있는 사용자는 새 팀 생성 및 합류 신청 불가

---

## 9. 화면 흐름 요약

### 9-1. 팀 생성 경로
프로필 등록 완료
→ 프로필 등록 완료 화면
→ [팀 생성/합류]
→ [팀 만들기]
→ 팀 정보 입력
→ 팀 생성 완료
→ 팀장/첫 팀원 자동 등록
→ 내 팀/팀 관리 화면

### 9-2. 팀 합류 경로
프로필 등록 완료
→ 프로필 등록 완료 화면
→ [팀 생성/합류]
→ [팀 찾기]
→ 팀 목록/검색
→ 팀 상세
→ [합류 신청]
→ 승인 대기
→ 팀장 승인
→ 내 팀 화면

---

## 10. 대회 참가와의 연결

대회 참가 단계는 팀 단계 이후에 이어진다.

즉,
- 팀 생성 완료 후 대회 참가 가능
- 팀 합류 승인 완료 후 대회 참가 가능

대회 참가 신청 주체는 우선 captain 기준으로 설계하는 것이 적절하다.

---

## 11. 현재 기준 결론

팀 단계는 `팀 생성 → 팀 합류` 순차가 아니라
`팀 생성 또는 팀 합류` 분기형 구조로 설계한다.

핵심 원칙:
- 프로필 등록 완료 후 팀 단계 진입
- 팀 생성자는 자동으로 captain + 팀원 등록
- 팀 합류는 신청 → 팀장 승인 방식
- 팀 미소속 사용자에게만 팀 생성/팀 찾기 노출
- 팀 소속 이후에는 내 팀/대회 참가 중심으로 전환
