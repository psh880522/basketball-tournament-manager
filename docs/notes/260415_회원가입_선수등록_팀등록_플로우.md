# 팀 생성 및 선수 등록 플로우 정리

## 1. 확정 흐름

회원가입 → 선수 등록 → 본인인증 → 팀 생성/합류 → 대회 참가

단,
UX에서는 `선수 등록 → 본인인증` 을 하나의 연속된 프로필 등록 스텝으로 취급한다.

즉 사용자 관점에서는:

- 회원가입
- 프로필 등록(선수 등록 + 본인인증)
- 팀 생성/합류
- 대회 참가

처럼 이해되도록 설계한다.

---

## 2. 완료 화면 기반 유도 흐름

### 2-1. 회원가입 완료 화면 추가
회원가입 직후 완료 화면을 노출한다.

목적:
- 가입 직후 이탈을 줄이고
- 바로 다음 행동인 선수 등록으로 유도

버튼:
- [건너뛰기]
- [선수등록]

동작:
- 선수등록 클릭 시 프로필 등록 플로우 진입
- 건너뛰기 클릭 시 홈/대시보드 등으로 이동

---

### 2-2. 프로필 등록 완료 화면 추가
선수 등록 + 본인인증 완료 후 완료 화면을 노출한다.

목적:
- 프로필 등록 완료를 명확히 인지시키고
- 다음 단계인 팀 생성/합류로 유도

버튼 예시:
- [건너뛰기]
- [팀 생성/합류]

또는 세분화 가능:
- [건너뛰기]
- [팀 만들기]
- [팀 찾기]

현재 의도 기준으로는
우선 `[건너뛰기] / [팀 생성/합류]` 형태로 단순하게 시작 가능

---

## 3. 단계 해석

### 3-1. 회원가입
- 계정 생성
- 서비스 기본 진입 가능

### 3-2. 프로필 등록
아래 두 단계를 하나의 흐름으로 묶음
- 선수 등록
- 본인인증

이 단계가 끝나면
- 개인 참가 자격 준비 완료 상태로 본다

### 3-3. 팀 단계
- 팀 생성
- 팀 찾기 / 팀 합류

### 3-4. 대회 참가
- 팀 단위 참가 신청
- 실제 대회 참여 흐름 진입

---

## 4. 건너뛰기 이후 재진입 동선

완료 화면에서 건너뛰기를 누르더라도
사용자가 나중에 쉽게 이어서 진행할 수 있어야 한다.

재진입 방식:

### 4-1. 사이드 메뉴 항목 추가
사이드 메뉴에 아래 항목 노출
- 선수등록
- 팀 생성
- 팀 찾기

조건에 따라 노출/강조 가능

예:
- 아직 프로필 등록 전이면 `선수등록` 강조
- 프로필 등록 완료 후 팀 미소속이면 `팀 생성`, `팀 찾기` 강조

### 4-2. 대시보드 상단 배너 노출
현재 사용자 진행 상태에 따라 다음 단계 배너 노출

예:
- 회원가입만 완료한 상태
  - "선수 등록을 완료하고 팀 활동을 시작해보세요"
- 본인인증까지 완료했지만 팀이 없는 상태
  - "팀을 만들거나 기존 팀에 합류해보세요"
- 팀까지 완료했지만 대회 참가 이력이 없는 상태
  - "참가 가능한 대회를 확인해보세요"

---

## 5. UX 설계 의도

이 흐름의 목적은 다음과 같다.

- 초기 가입 직후 사용자가 무엇을 해야 하는지 명확하게 제시
- 프로필 등록을 끝낸 뒤 팀 단계로 자연스럽게 이동
- 강제 완료를 최소화하면서도 다음 행동을 계속 유도
- 건너뛰더라도 메뉴와 배너를 통해 다시 이어갈 수 있게 구성

즉,
강한 퍼널 구조이면서도 완전 강제형은 아닌 유연한 온보딩 흐름으로 설계한다.

---

## 6. 현재 기준 정리된 화면 흐름

1. 회원가입 완료
2. 회원가입 완료 화면
   - [건너뛰기] / [선수등록]
3. 선수 등록
4. 본인인증
5. 프로필 등록 완료 화면
   - [건너뛰기] / [팀 생성/합류]
6. 팀 생성 또는 팀 찾기/합류
7. 대회 참가

---

## 7. 구현 시 참고 메모

- `선수 등록` 과 `본인인증` 은 내부적으로는 구분되더라도 UX상 하나의 프로필 등록 흐름으로 설계
- 완료 화면은 단순 성공 메시지가 아니라 다음 단계 CTA 역할 수행
- 건너뛰기 선택 시에도 기능 접근 경로를 잃지 않도록 메뉴/배너 제공
- 향후 사용자 상태 기반으로 대시보드 CTA를 동적으로 제어 가능
